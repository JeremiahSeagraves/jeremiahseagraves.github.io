What this interactive diagram represents

A production-style distributed backend that Jeremiah Seagraves designs and operates. Requests travel left to right and back: Clients to Load Balancer to API Gateway to one of several auto-scaling Spring (Java) microservices, then to a data tier of MongoDB (primary store), Redis (cache) and Kafka (event bus). Raising traffic adds microservice instances automatically; failing a node reroutes traffic to healthy instances with zero dropped requests, demonstrating zero-downtime resilience.

  • Load Balancer — distributes traffic and health-checks instances; reflects CI/CD and deploy-reliability work at Nordstrom.
  • API Gateway — REST and gRPC entrypoints; reflects APIs that onboarded 14,000+ restaurants at OpenTable and gRPC tax-validation services at Expedia.
  • Spring Microservices — Java 11–21 services with idempotency, retries and dependency injection; the core of every role held.
  • MongoDB — real-time Change Stream sync with resume tokens and distributed locking; cut sync latency 85% at 99.9% uptime at Optum.
  • Redis — low-latency caching layer across OpenTable, Nordstrom and Expedia.
  • Kafka — event streaming for cross-service consistency at OpenTable, Expedia and Optum.
